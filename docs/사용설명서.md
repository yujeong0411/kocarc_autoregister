# KOCARC 환자 자동등록 프로그램 — 사용설명서

엑셀에 환자 정보를 정리해두면, 프로그램이 KOCARC eCRF 사이트에 자동으로
**로그인 → 환자등록 → 8개 영역 입력 → 저장**을 해줍니다.

---

## ★ 가장 쉬운 방법: 실행파일(.exe) — 다른 사람도 이렇게 쓰면 됩니다

파이썬 설치가 전혀 필요 없습니다.

1. **준비물**: `KOCARC자동등록.exe` 파일 + 사용할 PC에 **Google Chrome** 설치.
2. `KOCARC자동등록.exe` **더블클릭** → 창이 뜹니다.
3. 창에서:
   - **① 입력 엑셀**: `빈 양식 만들기` 로 엑셀을 만들고, 환자 정보를 채운 뒤 `찾기…` 로 그 파일 선택.
   - **② 로그인**: 아이디·비밀번호 입력 (주소는 기본값 그대로).
   - **③ 모드**: 처음엔 **검증**(저장 안 함)으로! 사이트에 만든 연습환자 `PAT_ID` 를 넣고 확인.
   - **▶ 시작** 클릭 → 아래 로그창에서 진행 상황 확인. 문제 없으면 **실전**으로 바꿔 다시 실행.
4. 진행기록(`progress.csv`)은 exe 와 같은 폴더에 생깁니다.

> exe 는 `dist\KOCARC자동등록.exe` 에 있습니다. 이 파일 하나만 복사해 주면 됩니다.
> (배포·재빌드 방법은 맨 아래 참고)

## 0. 구성 파일

| 파일 | 설명 |
|---|---|
| **`KOCARC자동등록.exe`** | **창(GUI) 실행파일 — 더블클릭으로 사용** (`dist\` 폴더) |
| `kocarc_gui.py` | 창(GUI) 프로그램 소스 |

| 파일 | 설명 |
|---|---|
| `schema_extract.py` | 저장된 사이트 HTML → 폼 구조(`schema.json`) 추출 |

| `kocarc_bot.py` | **자동입력 봇** (로그인·등록·저장) |
| `KOCARC_입력양식.xlsx` | 환자 데이터를 적는 엑셀 (자동 생성됨) |
| `config.example.ini` | 설정 견본 (복사해서 `config.ini` 로 사용) |
| `pyproject.toml` | uv 프로젝트 정의 (의존성 목록) |

## 1. 설치 (최초 1회)

이 프로젝트는 **uv** 로 관리합니다.

1. **Google Chrome** 설치되어 있어야 합니다.
2. 명령창(PowerShell)에서 이 폴더로 이동 후:

   ```
   uv sync
   ```
   → `.venv` 가 자동으로 만들어지고 필요한 패키지가 설치됩니다.

> 이후 모든 명령은 `uv run python ...` 형태로 실행합니다. (가상환경을 따로 켤 필요 없음)

---

## 2. 엑셀 양식 만들기

이미 `KOCARC_입력양식.xlsx` 가 있으면 그대로 쓰면 됩니다.
다시 만들려면:

```
uv run python schema_extract.py
uv run python build_template.py
```

### 엑셀 작성법
- **[환자목록]** 시트: 환자 1명 = 1줄. `환자키`(1,2,3…)는 미리 들어가 있습니다.
- **각 영역 시트**(공통영역·예방역학…): 같은 `환자키` 줄에 값을 채웁니다.
- **[코드북]** 시트: 각 항목에 어떤 값을 넣어야 하는지 정리돼 있습니다.
  - 값은 **코드(M)** 또는 **의미(Male)** 둘 다 인식합니다. (코드북 표기 그대로 쓰는 게 안전)
- 1행=질문, **2행=필드명(프로그램이 읽는 줄, 수정 금지)**, 3행부터 데이터.
- 해당 없는 칸은 **비워두면** 됩니다.
- 날짜는 `2026-06-15` 형식. 시각은 `_HOUR`(시)·`_MIN`(분) 칸에 각각.

---

## 3. 설정 파일 만들기

`config.example.ini` 를 복사해서 **`config.ini`** 로 저장하고 값을 채웁니다.

- `login_url` : **로그인 화면 주소**(브라우저 주소창 그대로). ★꼭 확인
- `member_id` : 로그인 아이디
- `password`  : 비워두면 실행할 때 물어봅니다 (권장)
- `dry_run`   : `true`=저장 안 함(연습) / `false`=실제 저장

> ⚠️ `config.ini` 와 엑셀에는 환자 정보·비밀번호가 들어갑니다. **공유/업로드 금지.**

---

## 4. 실행 — 반드시 이 순서로

### ① 검증 (안전, 저장 안 됨)
실제 저장 없이 “값이 제대로 들어가는지”만 눈으로 확인합니다.

1. 사이트에 **연습용 환자 1명을 손으로** 만들고, 그 **PAT_ID**(예: `14-99999`)를 확인.
2. `config.ini` 에서 `dry_run = true`, `test_pat_id = 14-99999` 로 설정.
3. 엑셀에서 그 환자(환자키 1)의 데이터를 채웁니다.
4. 실행:
   ```
   uv run python kocarc_bot.py
   ```
5. 브라우저가 뜨고 각 영역이 자동으로 채워지는지 **눈으로 확인** (저장은 안 됨).
   - 빨간 경고 `! 확인필요` 로그가 나오면 그 칸의 값/코드를 점검하세요.

### ② 실전 (실제 저장)
1. `config.ini` 에서 `dry_run = false`, `test_pat_id` 는 비웁니다.
2. **처음엔 1명만**: `only_keys = 1` 로 두고 실행 → 사이트에서 결과 확인.
3. 문제 없으면 `only_keys` 를 비우고 전체 실행:
   ```
   uv run python kocarc_bot.py
   ```
- 진행 상황은 `progress.csv` 에 기록됩니다. 중간에 멈춰도 **다시 실행하면 이어서** 진행합니다(이미 완료한 환자는 건너뜀).

## 5. 주의 / 한계

- **자동저장은 실제 연구 DB에 기록**됩니다. 항상 검증(①) 후 소량(②)부터 진행하세요.
- 코드북에 없는 한글 표현(예: ‘남자’)은 인식 못 할 수 있으니 **코드북 표기**를 쓰세요.
- ‘소생 후 단계’의 **파일 첨부**는 자동화에 포함되지 않습니다(값 입력만).
- 조건부로 나타나는 항목도 값만 있으면 채워 넣습니다. 사이트 화면 구조가 바뀌면 조정이 필요할 수 있습니다.

---

## 6. 문제가 생기면

- `! 확인필요` 로그의 항목명을 코드북과 대조 → 엑셀 값 수정.
- 로그인 실패: 주소·아이디·비밀번호 확인.
- PAT_ID 확인 실패: 등록은 됐는지 사이트에서 보고, 실패 환자는 `progress.csv` 에 `no_pat_id` 로 남습니다.
- exe 가 “창만 뜨고 안 됨”: 그 PC에 **Chrome** 이 설치돼 있는지 확인 (필수). 인터넷 연결도 필요.

## 7. 배포 / 재빌드 (개발용)

### 다른 사람에게 줄 때
- `dist\KOCARC자동등록.exe` **이 파일 하나만** 복사해 주면 됩니다.
- 받는 사람은 **Chrome 설치 + exe 더블클릭**. 엑셀 양식은 프로그램의 `빈 양식 만들기` 로 생성.

### 코드 수정 후 exe 다시 만들기
사이트 화면이 바뀌거나 코드를 고치면:

```
uv run python schema_extract.py     # (HTML 다시 저장한 경우)
uv run pyinstaller --noconfirm KOCARC자동등록.spec
```
또는 처음부터:
```
uv run pyinstaller --noconfirm --onefile --windowed --name "KOCARC자동등록" ^
  --add-data "schema.json;." --collect-all selenium kocarc_gui.py
```
결과물: `dist\KOCARC자동등록.exe`

> ⚠️ exe 는 만든 OS(Windows)에서만 동작합니다. Mac 용은 Mac에서 따로 빌드해야 합니다.
